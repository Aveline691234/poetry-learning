## 一、产品概述

### 1.1 产品简介

**产品名称**：AI古诗小老师

**核心定位**：一款基于AI大模型的智能古诗学习伙伴，通过自然对话交互和语音技术，为儿童提供沉浸式、个性化的古诗背诵学习体验。

**当前版本支持功能**：
- 诗词学习（index_optimized.html）- 完整的学习流程
- 背诵挑战（recite.html）- 独立背诵训练
- 趣味游戏（game.html）- 字词排序游戏
- 摄像头互动（camera.html）- 动态字词捕捉
- AI聊天（chat.html）- 智能问答助手
- 学习进度（progress.html）- 进度追踪

### 3.1 核心功能矩阵

| 功能模块 | 功能名称 | 文件 | 优先级 | 状态 |
|---------|---------|------|--------|------|
| 诗词学习 | 完整学习流程 | index_optimized.html | P0 | ✅ 已完成 |

| 学习进度 | 进度追踪 | progress.html | P2 | ✅ 已完成 |

### 3.2 诗词学习模块 (index_optimized.html)

**功能描述**：完整的古诗学习流程，包含跟读、背诵、游戏等

**核心功能**：
1. **AI对话讲解**
   - 诗意解析
   - 作者介绍
   - 名句赏析
   - 写作背景

2. **跟读背诵**
   - AI范读
   - 语音识别
   - 发音评估
   - 自动进入下一句

3. **学习进度追踪**
   - 星级评分
   - 进度百分比
   - 本地存储

4. **趣味游戏**
   - 诗句填空
   - 字词排序

**技术特性**：
- 语音合成（Web Speech API）
- 语音识别（Web Speech API）
- AI对话（DeepSeek API）
- 本地存储（localStorage）

### 3.3 背诵挑战模块 (recite.html)

**功能描述**：独立的背诵训练模块，提供三种熟练程度选择

**核心功能**：
1. **熟练程度选择**
   - 🌱 初学者（30%熟悉）- 多提示引导
   - 🌿 进阶者（70%熟悉）- 适当提示
   - 🌳 熟练者（100%熟悉）- 自主挑战

2. **自动背诵流程**
   - AI语音引导
   - 自动聆听
   - 自动判断正误
   - 自动进入下一句

3. **智能提示系统**
   - 第一层提示：描述性提示
   - 第二层提示：第一个字
   - 第三层提示：前两个字
   - 第四层：诗意讲解

4. **DeepSeek AI集成**
   - 自然语言引导
   - 个性化鼓励
   - 背诵限制规则

**背诵限制规则**（来自index_optimized.html）：
- 永远不要直接说出完整诗句
- 永远不要说超过两个字的开头
- 不要把诗句用引号框起来
- 闲聊时必须引导回背诵主题

**技术特性**：
- 语音选择器（支持Xiaoxiao等语音）
- 延时播放（确保语音加载完成）
- 自动麦克风管理
- 语音与聆听状态互斥

### 3.4 趣味游戏模块 (game.html)

**功能描述**：通过字词排序组成完整诗句

**核心功能**：
1. 随机排列诗句中的字
2. 点击选择字词
3. 自动组成诗句
4. 正确性判断
5. 提示功能

### 3.5 摄像头互动模块 (camera.html)

**功能描述**：通过摄像头捕捉飘动的汉字

**核心功能**：
1. 摄像头视频流
2. 动态生成飘动汉字
3. 点击/拖动捕捉汉字
4. 计时计分系统
5. 正确性判断

### 3.6 AI聊天模块 (chat.html)

**功能描述**：与AI进行自由对话

**核心功能**：
1. 文字输入
2. 语音输入
3. AI回复
4. 语音朗读
5. 语音选择

### 3.7 学习进度模块 (progress.html)

**功能描述**：展示学习进度和历史

**核心功能**：
1. 当前学生信息
2. 已学诗句统计
3. 学习记录列表
4. 星级展示

## 四、非功能需求

### 4.1 性能需求

| 指标 | 要求 | 说明 |
|------|------|------|
| 页面加载时间 | <3秒 | 首次加载 |
| AI响应时间 | <3秒 | 正常网络环境 |
| 语音识别延迟 | <500ms | 实时性要求 |
| 动画帧率 | ≥30fps | 流畅体验 |

### 4.2 兼容性需求

| 类型 | 要求 |
|------|------|
| 浏览器 | Chrome 90+, Edge 90+, Safari 14+ |
| 操作系统 | Windows 10+, macOS 10.15+, iOS 13+, Android 8+ |
| 网络 | 支持4G/5G/WiFi |
| 硬件 | 需麦克风（语音功能）、摄像头（可选） |

### 4.3 安全需求

| 需求 | 说明 |
|------|------|
| HTTPS | 生产环境必须使用HTTPS |
| API Key保护 | DeepSeek API Key存储在后端环境变量 |
| CORS | 前端通过后端代理调用API |

## 五、部署需求

### 5.1 部署方案

**方案一：Vercel + 后端服务**
- 前端：Vercel（静态托管）
- 后端：Railway/Render（Python服务）

**方案二：全栈部署**
- Vercel Serverless Functions（Python）

### 5.2 环境变量

| 变量名 | 说明 | 必填 |
|--------|------|------|
| DEEPSEEK_API_KEY | DeepSeek API密钥 | 是 |

### 5.3 部署文件结构

```
poetry-learning/
├── api/
│   └── chat.py          # Vercel Serverless函数
├── vercel.json          # Vercel配置
├── requirements.txt      # Python依赖
├── index_optimized.html # 诗词学习
├── recite.html          # 背诵挑战
├── game.html            # 趣味游戏
├── camera.html          # 摄像头互动
├── chat.html            # AI聊天
└── progress.html        # 学习进度
```
